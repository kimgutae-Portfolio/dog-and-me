# 1次開発の範囲

## 目的

最初の開発では、AI動画生成の自動化より先に次の3点を検証します。

1. 日本のお客様にサービスの価値と安心感が伝わるか
2. 感情的な負担なく、愛犬の話と制作希望を入力できるか
3. 受注後の確認プロセスが見え、安心して待てるか

## 今回実装した範囲

| 領域 | 状態 | 内容 |
| --- | --- | --- |
| ブランドサイト | 完了 | コンセプト、3つの用途、作品トーン、制作フロー、料金、FAQ |
| 相談フォーム | 完了 | 5ステップ、必須項目、センシティブな内容の除外指定、自動保存 |
| 写真選択 | UI完了 | 最大20枚の選択とファイル名確認。外部送信なし |
| 制作室 | デモ完了 | 受付内容、進行ステータス、完成までの流れ |
| レスポンシブ | 完了 | PC、タブレット、モバイル |
| 認証 | 次期 | Supabase Auth（メール、Google、Apple） |
| DB・Storage | 次期 | 注文、回答、素材、進行状態の永続化とprivate bucket |
| 決済 | 次期 | Stripe Checkout + webhook |
| 管理画面 | 次期 | 注文一覧、担当者、ステータス、納期、素材確認 |
| 動画制作 | 手動運用 | 外部ツールで制作し、完成品を管理者がアップロード |
| AI自動生成 | 範囲外 | 品質検証後の第2段階 |

## 1次運用の成立条件

- 相談受付時点では課金せず、担当者が素材と納期を確認する
- 写真はprivate storageへ直接アップロードし、アプリサーバーを経由させない
- AI生成結果をお客様へ自動公開しない
- メモリアル用途では病院、死因、最後の瞬間を明示的な許可なく扱わない
- お客様の明示的な同意なく写真・動画をモデル学習へ使わない

## 次のスプリント

1. Supabaseプロジェクトと環境変数を接続
2. Auth、profiles、pets、orders、order_answersを実装
3. Signed uploadでassetsを保存
4. 管理者注文詳細とステータス変更を実装
5. Stripe Checkoutを接続
6. メール通知を接続
7. 実注文を3〜5件手動制作し、品質と工数を計測

AI Provider連携とFFmpeg自動レンダリングは、この運用データを確認した後に進めます。
